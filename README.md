# StepRead

最近做了个 Edge 浏览器插件 StepRead，主要是想帮自己学习硬核书籍。

它支持在浏览器里打开 PDF，选中看不懂的句子后直接在旁边问 AI，并且可以根据连续追问形成知识图谱。这样的学习流当然也可以在普通 AI 网站里完成，只是那里很容易把不同书、不同问题、不同任务混到同一个上下文里。StepRead 想解决的是一个更具体的问题：把提问、上下文、页码、划线和后续追问都留在当前 PDF 的阅读现场里。

它更适合结构精致的 PDF：有文字层、有目录、有比较稳定的段落和页码。没有 OCR 的扫描版 PDF 暂时不太行。出于自己的产品品味，我把用户能控制的参数基本都塞进设置页面了，大家可以下载体验一下。

## 核心功能

- 在 Edge 浏览器中打开 PDF，并进入 StepRead 阅读器。
- 阅读器显示 PDF 正文、当前页码、目录/书签，并支持目录跳转。
- 选中文本后划线提问。
- 根据选中文本、当前位置和上下文调用你配置的 AI 模型。
- 支持围绕同一段文本继续追问，形成连续阅读线索。
- 在知识图谱页面查看问题、回答、引用文本和追问关系。
- 在设置页调整模型、接口地址、上下文范围、主题、划线样式、阅读器布局和交互参数。

## 适合什么 PDF

StepRead 依赖 PDF 的文字层和结构信息。推荐使用：

- 正版电子书、论文、报告、课程讲义等文字层清晰的 PDF。
- 有目录、标题层级、页码结构比较稳定的 PDF。
- 复制文字时不会严重乱码的 PDF。

暂时不推荐：

- 纯扫描图片 PDF。
- 没有 OCR 的影印版 PDF。
- 文字层顺序混乱、复制出来大量错位或乱码的 PDF。

## 本地安装方式

这是一个可以本地加载的 Edge 扩展。下载后不需要编译，直接把解压后的文件夹加载到 Edge 即可。

1. 下载本仓库。
   - GitHub 页面点击 `Code` -> `Download ZIP`。
   - 或者用 Git 克隆本仓库。
2. 解压到一个固定目录。
   - Windows 推荐：`D:\Tools\StepRead-extension`
   - 也可以放在：`C:\Users\<你的用户名>\Documents\StepRead-extension`
   - 不建议放在临时目录，也不要加载后随手删除或移动。Edge 加载的是这个目录本身。
3. 打开 Edge 扩展管理页。
   - 地址栏输入：`edge://extensions/`
4. 打开“开发人员模式”。
5. 点击“加载解压缩的扩展”。
6. 选择刚才解压后的 StepRead 文件夹。
   - 这个文件夹的根目录里应该能看到 `manifest.json`。
   - 不要选择 `src` 文件夹。
   - 不要直接选择压缩包。
7. 加载完成后，可以把 StepRead 固定到浏览器工具栏。

以后更新时，把本地文件夹替换成新版本，然后回到 `edge://extensions/`，在 StepRead 卡片上点击“重新加载”。

## AI 配置

StepRead 使用你自己配置的模型接口，也就是 BYOK（Bring Your Own Key）。

1. 打开 Edge 扩展管理页：`edge://extensions/`
2. 找到 StepRead。
3. 点击“详细信息”或“扩展选项”。
4. 在设置页填写模型参数，例如：
   - API Base URL
   - API Key
   - 模型名称
   - 上下文范围
   - 回答、知识图谱、划线和阅读器相关参数
5. 保存设置后再回到阅读器使用。

不要把自己的 API Key 写进代码、截图或提交到 GitHub。设置页保存的密钥只应该留在你自己的浏览器扩展存储里。

## 使用流程

1. 在 Edge 中打开一个 PDF。
2. 点击 StepRead 扩展按钮，把当前 PDF 打开到 StepRead 阅读器。
3. 在阅读器中选中文本。
4. 输入问题并发送。
5. 查看 AI 回答。
6. 根据回答继续追问，或者回到正文继续划线。
7. 打开知识图谱页面，查看已经形成的问题、回答和文本引用关系。

如果某个 PDF 无法正确读取、目录跳转不稳定，通常原因是 PDF 自身的文字层、目录或页码结构不稳定。可以先测试一份文字层干净的 PDF，确认插件流程是否正常。

## 数据和隐私边界

- PDF 文件来自用户本地或浏览器当前打开的页面。
- 阅读记录、划线、问答和设置主要保存在浏览器扩展自己的本地存储中。
- 只有在你主动提问、生成回答或使用 AI 相关功能时，StepRead 才会把选中文本和必要上下文发送到你配置的 AI 服务。
- 本仓库不包含我的本地 PDF、阅读记录、API Key 或本地路径。

## 常见问题

### 为什么扫描版 PDF 不好用？

StepRead 主要读取 PDF 的文字层。扫描版 PDF 的页面本质上是图片，如果没有 OCR 文字层，插件无法稳定知道你选中的文字是什么，也无法可靠建立页码、段落和目录关系。

### 为什么有些 PDF 目录跳转不准？

PDF 的目录项通常指向内部位置对象。不同 PDF 的目录、页码、坐标和文字层质量差异很大。StepRead 会尽量根据页码和标题位置修正，但低质量结构仍可能导致跳转偏移。

### 可以直接上传压缩包到 Edge 吗？

本地开发安装时不要选择压缩包。Edge 的“加载解压缩的扩展”需要选择解压后的文件夹，而且这个文件夹根目录必须包含 `manifest.json`。

### 可以发布到 Edge 扩展商店吗？

可以，但商店发布需要额外准备商店描述、截图、隐私说明、权限说明和最终 zip 包。本仓库主要用于公开代码和让用户本地安装体验。
