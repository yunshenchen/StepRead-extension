StepRead 设置

这里集中管理模型调用、context 接入和阅读器外观。密钥只保存到本机浏览器扩展存储。

基础 AI

第一次使用时，通常只需要在这里填写模型接口、API 密钥和模型名称。知识图谱 prompt 在知识图谱页面按次填写，不在设置页保存长期版本。

默认情况下，划线问答和知识图谱都会复用这里的模型。

AI context 接入

context 是每次请求 AI 时，随问题一起发送给模型的资料包。这里分开控制“划线问答”和“知识图谱”两套资料包，可以选择原文、目录、历史划线和历史问答；内容越完整，回答依据越充分，但响应会更慢，也更容易接近模型文本上限。

当前文档数据状态

正在读取当前文档数据状态；这里展示的是记录数量，不是模块开关。

划线问答 context

用于回答某一条划线旁边的问题。建议先保留段落和少量相邻段落；只有问题需要更长推理链时，再打开章节全文或目录结构。

原文范围

控制 AI 能看到的正文边界

这些选项决定从划线附近取多少原文，越往后文本量越大。

目录结构

让回答知道当前位置和全文结构

目录适合定位章节关系和当前阅读位置。即使当前文档暂时没有结构数据，这项也可以继续保存；生成 context 时只会读取已有目录。

历史问答

决定是否复用之前的追问记录

连续追问适合保持当前对话上下文；历史问答适合复用同一文档里已经澄清过的信息。

知识图谱 context

用于生成整篇文档的学习结构，不针对某一条划线回答。这里更适合接入全局证据和目录结构，但全文仍建议作为最后的兜底选项。

阅读证据

选择知识图谱的核心材料

划线和问答是你已经关注过的内容，通常比直接接入全文更稳定。

局部原文

为划线补充必要的上下文

这些设置会围绕每条划线补充正文段落，适合减少孤立摘句带来的误解。

目录结构

决定知识图谱的层级依据

目录负责章节层级和当前位置。缺少真实标题结构时，目录设置仍可保存；生成 context 时只会读取当前文档已有的结构数据。

全文兜底

仅在需要全篇扫描时开启

全文会显著增加 token 占用。长文档建议先用划线、问答和目录生成，再视结果决定是否扩大范围。

阅读器外观

如果正文阅读不舒服、字体偏小，或想切换浅色/深色主题，可以修改这里。
